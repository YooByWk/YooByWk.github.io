# NestJS

`NestJS` 는 Node.js 서버 애플리케이션을 구축하기 위한 프레임워크.

`TypeScript`를 주력으로 사용하며, 엔터프라이즈급 애플리케이션을 위해 여러 개념(OOP, FP, FRP) 개념이 통합됨

## 철학

> Spring 과 유사하다.

NestJS는 Angular의 구조적 철학과 Java Spring의 엔터프라이즈급 설계 패턴을 Node.js 환경에 도입하는 것을 목표로 한다.

### 1. 의존성 주입 (DI, Dependency Injection)

**정의:** 클래스가 자신이 의존하는 객체를 직접 생성하는 대신, IOC (Inversion of Control) 컨테이너가 관리하는 객체를 주입받아 사용

**OOP 연관성** : DI는 SOLID 원칙 중 **DIP (의존성 역전 원칙)** 를 구현하는 핵심 메커니즘이다. 이를 통해 구성 요소 간의 결합도를 낮추고 테스트 용이성을 극대화한다.

### 2. 모듈성 (Modularidad)

**정의:** 애플리케이션을 Module 단위로 분리하고 캡슐화하여 관리

**의의:** 각 모듈은 관련된 **Controller** 와 **Provider** 를 묶어 책임 범위를 명확히 하며, 대규모 애플리케이션의 구조적 복잡성을 효과적으로 관리할 수 있게 한다.

### 3. AOP (관점 지향 프로그래밍) 및 데코레이터

**정의:** 데코레이터(@)를 사용하여 핵심 비즈니스 로직과 **부가 기능(횡단 관심사)** 을 분리

**활용:** `@UseGuards` (인증/인가), `@UseInterceptors` (응답 변환/로깅), `@UsePipes` (유효성 검사) 등을 통해 코드를 구조적이고 선언적으로 유지한다.

## 핵심 요소

1. `Controller` : 클라이언트의 Request를 처리하고 Response를 반환하는 계층
2. `Provider(service)` : 복잡한 비즈니스 로직을 처리하는 계층, DI 컨테이너에 등록되어 사용
3. `Module` : Controller 와 Provider를 묶어 애플리케이션의 구조적 단위를 정의

## 특이사항

- `Express.js`에 비해 구조가 복잡함 : 오히려 구조화 된 장점이기도 함
- 공식문서가 잘 되어있음. 어지간한 정보는 공식문서에 다 나와있다고 봐도 무방
- 마이크로 서비스 지원 : HTTP, gRPC, MQTT, Redis 외 다양한 마이크로서비스 통신 방식을 쉽게 통합할 수 있는 모듈을 제공해줌...
